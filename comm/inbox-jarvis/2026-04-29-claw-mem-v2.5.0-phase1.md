## Mission

Implement **Semantic Search Integration** for claw-mem v2.5.0.

---

## Background

claw-mem currently uses BM25 + Heuristic retrieval. This phase adds embedding-based semantic search to improve recall accuracy.

---

## Requirements

### P0 - Must Have

#### 1. Semantic Search Integration

- Add support for embedding-based semantic search
- Support multiple embedding models:
  - OpenAI text-embedding-3-small
  - Local models (sentence-transformers)
- Hybrid search: combine BM25 + semantic scores
- Maintain backward compatibility

#### 2. Memory Compression V2

- Improve F5 compression algorithm
- Better quality-to-size ratio
- Preserve semantic meaning

### P1 - Should Have

#### 3. Search Ranking Improvements

- Improve BM25 + Heuristic hybrid ranking
- Weight tuning based on query type

#### 4. Performance Optimization

- Target: <5ms avg retrieval latency
- Cache embedding results

---

## Technical Approach

### Semantic Search Architecture

```
User Query → Embedding Model → Vector DB Search → Hybrid Score → Results
                ↓
         (Optional: Cache)
```

### Key Components

1. **EmbeddingService** - Manages embedding models
2. **VectorAdapter** - Interface for vector databases
3. **HybridSearcher** - Combines BM25 + semantic scores
