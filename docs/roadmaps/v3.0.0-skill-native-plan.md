# claw-mem v3.0.0 规划: Skill 原生化

## 目标

将 claw-mem 从 "Python 包 + Bridge 外部服务" 模式改造为原生 OpenClaw Skill 架构,与 ResearchClaw 保持一致.

## 背景

当前状态:
- claw-mem v2.2.0 使用 `plugins.slots.memory` 槽位集成
- 需要独立运行 `bridge.py` 进程
- 没有完整的 Skill 生命周期钩子

目标状态 (参考 ResearchClaw):
- 完整的 `BaseSkill` 抽象类实现
- 原生命令注册 (`/claw-mem`, `/memory` 等)
- 生命周期钩子 (`on_load`, `on_enable`, `on_disable`, 等)
- 即插即用,放入 `~/.openclaw/skills/` 即可工作

### 核心接口设计

参考 `ResearchClawSkill`:

```python
from claw_mem import MemoryManager

class BaseMemorySkill(ABC):
    @abstractmethod
    def recall(self, query: str, **kwargs) -> List[MemoryResult]:
        pass
    
    @abstractmethod
    def memorize(self, content: str, **kwargs) -> bool:
        pass
    
    @abstractmethod
    def forget(self, memory_id: str) -> bool:
        pass

class ClawMemSkill(BaseMemorySkill):
    def __init__(self, config: Dict):
        self.manager = MemoryManager(config)
```

### 生命周期钩子

| 钩子 | 描述 |
|------|------|
| `on_load()` | 初始化 MemoryManager,加载索引 |
| `on_enable()` | 启用自动捕获/召回 |
| `on_disable()` | 暂停自动捕获 |
| `on_unload()` | 保存状态,清理资源 |

### 命令设计

| 命令 | 描述 |
|------|------|
| `/memory recall <query>` | 搜索记忆 |
| `/memory store <content>` | 存储记忆 |
| `/memory list` | 列出最近记忆 |
| `/memory stats` | 查看统计 |
