## 核心贡献

> "我们提出 HISA,将搜索路径从平面 token 扫描重写为两阶段层次化过程."

**关键结果**:
- 64K 上下文: **3.75×** 加速
- 无需额外训练
- 保持 DSA 相同的 token 级稀疏模式

---

## 问题: Token 级稀疏注意力的瓶颈

### DSA (DeepSeek Sparse Attention)

```
Indexer 对每个 query:
  1. 扫描整个前缀 (L tokens)
  2. 计算每个 token 的相关性分数
  3. 选择 top-k tokens
  4. 传递给 Sparse MLA

瓶颈:
  每层索引成本 = O(L²)
  与稠密注意力相同渐近复杂度!
```

---

## HISA 两阶段层次化索引

```
┌─────────────────────────────────────────────────────────────┐
│                    HISA 层次化索引                           │
│                                                             │
│  Stage 1: Block-Level Coarse Filtering                      │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  1. 将前缀划分为大小 B 的块                              │  │
│  │  2. 对每个块计算池化表示向量 (mean pooling)              │  │
│  │  3. Query 对所有块代表评分                               │  │
│  │  4. 保留 top-m 块                                       │  │
│  │                                                       │  │
│  │  成本: O(L/B)                                          │  │
│  └──────────────────────────────────────────────────────┘  │
│                           ↓                                  │
│  Stage 2: Token-Level Refinement                            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  1. 仅在候选块内应用原始 DSA indexer                      │  │
│  │  2. 从 mB 个候选 token 中选择 top-k                      │  │
│  │                                                       │  │
│  │  成本: O(mB)                                           │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
│  总成本: O(L/B + mB) vs 原始 O(L)                           │
│  每层成本: O(L²/B + LmB) vs 原始 O(L²)                       │
└─────────────────────────────────────────────────────────────┘
```

## 核心创新

### 1. Plug-and-Play 替换

- 无需重新训练
- 无需修改注意力架构
- 无需修改 KV cache 布局

### 2. 相同输出结构

- 输出: 每个 query 一组 k 个 token 索引
- 下游 Sparse MLA 完全不变

### 3. 硬件友好

- 块级操作映射到 GPU 分块矩阵乘法
- TileLang GPU kernel 优化

### 层次化记忆检索

```python
# claw-mem v2.1.0
class HierarchicalMemoryIndex:
    """层次化记忆索引"""
    
    def __init__(self, block_size: int = 256):
        self.block_size = block_size
        self.blocks = []  # 块级索引
        self.tokens = {}  # Token 级索引
    
    def index(self, documents):
        """构建索引"""
        # 1. 划分块
        for doc in documents:
            for i in range(0, len(doc), self.block_size):
                block = doc[i:i+self.block_size]
                
                # 块级池化表示
                block_repr = self._pool(block)
                self.blocks.append(block_repr)
                
                # Token 级索引
                for j, token in enumerate(block):
                    self.tokens[(i, j)] = self._encode(token)
    
    def retrieve(self, query, k: int = 10):
        """两阶段检索"""
        # Stage 1: 块级粗过滤
        block_scores = [
            self._score(query, block) 
            for block in self.blocks
        ]
        top_blocks = self._top_k(block_scores, m=5)
        
        # Stage 2: Token 级精化
        candidates = []
        for block_idx in top_blocks:
            candidates.extend(
                self.tokens_in_block(block_idx)
            )
        
        token_scores = [
            (tok, self._score(query, tok))
            for tok in candidates
        ]
        
        return self._top_k(token_scores, k)
```

## 对 neoclaw 的启发

### 长上下文优化

```python
# neoclaw v2.1.0
class LongContextRetriever:
    """长上下文检索器"""
    
    def __init__(self):
        self.index = HierarchicalMemoryIndex()
    
    def retrieve_for_context(self, query, max_tokens: int):
        """为上下文检索"""
        # 使用 HISA 快速检索
        results = self.index.retrieve(query, k=max_tokens)
        
        # 返回用于上下文填充
        return [r.content for r in results]
```

## 总结

**HISA 的核心贡献**:
1. **两阶段索引**: 块级粗过滤 + Token 级精化
2. **Plug-and-Play**: 无需训练替换
3. **复杂度降低**: O(L²) → O(L²/B + LmB)
4. **质量保持**: 与 DSA 相近

**对 Project Neo 的意义**:
- claw-mem: 层次化索引
- neoclaw: 长上下文优化
