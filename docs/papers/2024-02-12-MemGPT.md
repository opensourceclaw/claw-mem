## 核心贡献

> "我们提出虚拟上下文管理，一种借鉴传统操作系统中分层记忆系统的技术，通过物理内存和磁盘之间的分页提供扩展虚拟内存的假象。"

**MemGPT**: 一个 OS 启发的 LLM 系统，智能管理不同存储层，在 LLM 有限上下文窗口内有效提供扩展上下文。

---

## 问题: 有限上下文窗口

### 上下文窗口限制

| 模型 | 上下文窗口 | 消息数 (约) |
|------|------------|-------------|
| Llama 2 | 4k | 60 |
| GPT-4 | 8k | 140 |
| GPT-4 Turbo | 128k | 2,600 |
| Claude 2 | 100k | 2,000 |

### 两个关键问题

1. **计算成本**: 直接扩展上下文长度导致二次方计算时间增长
2. **利用效率**: 长上下文模型难以有效利用额外上下文

## 核心组件

### 1. Queue Manager (队列管理器)

**职责**:
- 管理消息在 recall storage 和 FIFO queue 之间流动
- 控制上下文溢出
- 生成递归摘要

**内存压力警告**:
```
Warning Token Count (70% 上下文窗口)
  → 插入系统消息警告 LLM
  → LLM 使用函数将重要信息存储到 working context 或 archival storage

Flush Token Count (100% 上下文窗口)
  → 清空队列
  → 生成新的递归摘要
  → 驱逐的消息存储到 recall storage
```

### 2. Function Executor (函数执行器)

**核心函数**:

| 函数 | 描述 |
|------|------|
| `working_context.append()` | 添加信息到工作上下文 |
| `working_context.replace()` | 替换工作上下文中的信息 |
| `recall_storage.search()` | 搜索召回存储 |
| `archival_storage.insert()` | 插入到归档存储 |
| `archival_storage.search()` | 搜索归档存储 |

**自导向内存管理**:
- LLM 自主决定何时移动数据
- LLM 自主修改主上下文
- 无需用户干预

### 3. Function Chaining (函数链)

```python
# 多函数顺序调用
# 例如: 多页结果导航，多文档数据整理

# 特殊标志请求立即返回控制
function_call(request_heartbeat=True)  # 继续处理
function_call(request_heartbeat=False) # yield，等待外部事件
```

### 分层记忆实现

```python
# claw-mem v2.1.0
class MemGPTStyleMemory:
    """MemGPT 风格分层记忆"""
    
    def __init__(self, context_window: int = 8192):
        self.context_window = context_window
        self.warning_threshold = int(context_window * 0.7)
        self.flush_threshold = context_window
        
        # Main Context
        self.system_instructions = ""
        self.working_context = WorkingContext(max_tokens=2000)
        self.fifo_queue = FIFOQueue()
        
        # External Context
        self.recall_storage = RecallStorage()
        self.archival_storage = ArchivalStorage()
    
    def add_message(self, message: Message):
        """添加消息"""
        self.fifo_queue.append(message)
        self.recall_storage.store(message)
        
        # 检查内存压力
        current_tokens = self._count_tokens()
        
        if current_tokens > self.warning_threshold:
            self._send_memory_warning()
        
        if current_tokens > self.flush_threshold:
            self._flush_queue()
    
    def _send_memory_warning(self):
        """发送内存压力警告"""
        self.fifo_queue.append(SystemMessage(
            type="memory_pressure",
            content="Context approaching limit. Consider archiving."
        ))
    
    def _flush_queue(self):
        """清空队列"""
        evicted = self.fifo_queue.evict(int(self.context_window * 0.5))
        summary = self._generate_recursive_summary(evicted)
        self.fifo_queue.set_recursive_summary(summary)
```

## 对 neoclaw 的启发

### OS 启发设计

MemGPT 的 OS 启发设计可直接应用于 neoclaw:

```python
# neoclaw v2.1.0
class NeoclawMemoryManager:
    """neoclaw 内存管理器 (MemGPT 启发)"""
    
    def __init__(self):
        # 分层记忆
        self.main_context = MainContext()
        self.external_context = ExternalContext()
        
        # 管理器
        self.queue_manager = QueueManager()
        self.function_executor = FunctionExecutor()
    
    def process_event(self, event: Event):
        """处理事件"""
        # 1. 解析事件
        message = self._parse_event(event)
        
        # 2. 添加到队列
        self.queue_manager.append(message)
        
        # 3. 触发 LLM 推理
        output = self._inference()
        
        # 4. 执行函数调用
        for func_call in self._parse_output(output):
            result = self.function_executor.execute(func_call)
            self.queue_manager.append(result)
```

## 总结

**MemGPT 的核心贡献**:
1. **虚拟上下文管理**: 通过分页提供无限上下文假象
2. **分层记忆**: Main Context + External Context
3. **自导向内存管理**: LLM 自主管理内存
4. **函数链**: 多步检索和整理

**对 Project Neo 的意义**:
- claw-mem: 分层记忆 + 内存压力警告
- neoclaw: OS 启发内存管理架构
