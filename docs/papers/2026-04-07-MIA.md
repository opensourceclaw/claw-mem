## 核心贡献

> "我们提出 MIA，一个 Manager-Planner-Executor 架构，通过参数和非参数记忆的双向转换实现高效记忆演化。"

**关键结果**:

## 问题: 长上下文记忆的局限

### Deep Research Agents 的挑战

| 问题 | 描述 |
|------|------|
| **注意力稀释** | 长上下文阻碍模型理解当前问题 |
| **噪声引入** | 无关内容导致推理能力下降 |
| **存储挑战** | 持续增长的历史上下文 |
| **检索成本** | 大规模记忆检索计算成本高 |

### 知识导向 vs 过程导向记忆

| 类型 | 描述 | 示例 |
|------|------|------|
| **知识导向** | 结果是什么 | 用户属性、历史事实、检索文档 |
| **过程导向** | 结果如何获得 | 搜索轨迹、失败尝试、成功策略 |

## 核心创新

### 1. Manager-Planner-Executor 架构

- **解耦**: 历史记忆、参数规划、动态执行分离
- **解决**: 存储瓶颈和推理低效

### 2. 交替强化学习

- Planner 和 Executor 相互对齐
- 高层规划和底层检索协同优化

### 3. 测试时持续学习

- Planner 在推理过程中更新参数知识
- 不中断推理过程

### 4. 参数/非参数记忆双向转换

```
非参数记忆 (Manager) ←→ 参数记忆 (Planner)

压缩轨迹 ←→ 训练更新
历史洞察 ←→ 策略固化
```

### 过程导向记忆系统

```python
# claw-mem v2.1.0
class ProcessOrientedMemory:
    """过程导向记忆系统"""
    
    def __init__(self):
        # 非参数记忆 (Manager)
        self.episodic_memory = EpisodicMemory()
        
        # 参数记忆 (Planner)
        self.strategy_memory = ParametricMemory()
        
        # 双向转换
        self.converter = MemoryConverter()
    
    def store_trajectory(self, trajectory):
        """存储轨迹"""
        # 1. 压缩存储到情景记忆
        compressed = self._compress(trajectory)
        self.episodic_memory.store(compressed)
        
        # 2. 提取策略更新参数记忆
        insights = self._extract_insights(trajectory)
        self.converter.to_parametric(insights)
    
    def retrieve_for_planning(self, query):
        """为规划检索"""
        # 从参数记忆获取策略
        strategy = self.strategy_memory.get_strategy(query)
        
        # 从非参数记忆获取相似案例
        cases = self.episodic_memory.retrieve_similar(query)
        
        return strategy, cases
    
    def update_during_inference(self, new_experience):
        """测试时更新"""
        # 在线学习
        self.strategy_memory.update(new_experience)
```

### 交替强化学习

```python
# claw-rl v2.2.0
class AlternatingRLTrainer:
    """交替强化学习训练器"""
    
    def __init__(self, planner, executor):
        self.planner = planner
        self.executor = executor
    
    def train_alternating(self, experiences):
        """交替训练"""
        for exp in experiences:
            # 1. 固定 Planner，训练 Executor
            with freeze(self.planner):
                executor_loss = self._train_executor(exp)
            
            # 2. 固定 Executor，训练 Planner
            with freeze(self.executor):
                planner_loss = self._train_planner(exp)
            
            # 3. 协同优化
            self._joint_optimize(exp)
```

## 总结

**MIA 的核心贡献**:
1. **三组件架构**: Manager-Planner-Executor
2. **双向转换**: 参数 ↔ 非参数记忆
3. **交替 RL**: Planner 和 Executor 协同
4. **测试时学习**: 持续演化
5. **过程导向**: 策略而非仅知识

**对 Project Neo 的意义**:
- claw-mem: 过程导向记忆 + 双向转换
- claw-rl: 交替强化学习
